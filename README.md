# OSER

Open Source Editorial Renderer.

OSER is an early-stage open source project for turning structured Markdown and editorial text into durable publishing outputs: semantic HTML, editorial CSS, paginated HTML, EPUB, and PDF.

The goal is to provide a common rendering core for publishing projects that need predictable document structure, reusable editorial styling, and generated outputs that can be rebuilt from source text.

Current status: planning / seed repo.

## Problem

Many publishing workflows mix source text, presentation, layout, and export logic too early. That makes documents harder to version, review, reuse, and render consistently across web, print, and ebook outputs.

OSER starts from a simpler premise:

```text
Markdown / structured text
  -> document model
  -> semantic HTML
  -> editorial CSS
  -> PDF / EPUB / Web
```

The source text should remain the primary artifact. Rendered outputs should be derived artifacts that can be regenerated by the renderer.

## Scope

OSER is intended to become a renderer, not a writing UI.

Initial work will focus on:

- parsing structured Markdown into a document-oriented model
- producing semantic HTML
- defining editorial CSS conventions
- supporting paged preview and export workflows
- keeping integration points explicit for downstream tools

It will not start as a WYSIWYG editor, CMS, or visual page builder.

## Relationship To Other Projects

OSER is planned as the shared rendering foundation for:

- TRURL
- diegomadero.com
- editorial-ai-lab
- future publishing and editorial tooling

Those projects may provide UI, content workflows, site generation, automation, or product-specific behavior. OSER should remain focused on the rendering pipeline and the portable document representation beneath those outputs.
